# 运动项目输出模板

## 高密度摘要

- **一句话结论**：介绍 HIIT、动物流、跑步、瑜伽、壶铃等运动项目时，优先用“是什么、好处、风险、适用情况、怎么做”的轻量结构，而不是百科式长文。
- **核心机制**：先用一句话压缩项目定位，再分别交代收益、风险、适用边界和可执行做法，最后补引用与推荐资源，降低模型自由发挥。
- **判断入口**：当用户问“某个运动项目是什么、适不适合我、怎么开始练”时使用；如果用户要完整训练周期，再切换到训练计划模板。
- **常见误区**：不要把模板扩展成运动百科；“好处”和“风险”都只写一句话，重点放在适用场景和怎么做。
- **相关文档**：[技术索引](../../data/technology.md)、[Prompt 优先级](../../data/p/prompt-priority.md)、[Few Shot](../../data/p/prompt-fw-few-shots.md)。

## 模板

```text
请按以下 Markdown 结构介绍【运动项目】，所有一级内容都使用 h2。

## 一句话介绍
用一句话说明它是什么，以及主要训练什么能力。

## 好处
用一句话概括它最主要的收益。

## 风险
用一句话概括它最主要的风险，以及风险通常来自哪里。

## 适用情况
说明它适合什么目标、什么人、什么场景。

## 怎么做
分为入门版、标准版、进阶版。每版说明：
- 做什么
- 做多久
- 强度
- 频率
- 注意事项

## 不适用情况
说明哪些人或状态需要避免、谨慎或调整。

## 常见错误
列出 3-5 个常见错误，并给出修正方式。

## 引用
列出支撑关键判断的来源，包括研究、指南、教材、专业机构页面或高质量综述。
如果没有可靠来源，不要编造，写“暂无可靠引用”。

## 推荐资源
推荐继续学习或实践的资源，例如书籍、课程、视频、网站、App、专业机构或关键词。
优先推荐高质量、可操作、适合入门的资源。
```

## 使用说明

适合用于介绍具体运动项目，例如 HIIT、动物流、跑步、瑜伽、普拉提、壶铃、CrossFit、游泳、攀岩、拳击、跳绳等。

如果用户没有指定运动项目，把【运动项目】替换为用户问题中的具体项目；如果用户同时提到多个项目，应分别按同一结构输出，或先用对比模板判断哪一个更适合。

当用户要求个性化训练安排时，不要只套用本模板；应先询问或补充必要变量，包括训练目标、训练经验、伤病史、可用时间、器材条件和当前体能水平。
