# 拆书方法论
```md
拆书：{书名}

请完成以下任务：

1. 生成两个文件，并保存到 `data/b` 目录下：
   - `book-[english-name].md`
   - `book-[english-name].html`

2. 必须先读取并遵循这个设计文件：
   - `knowledge/b/book-breakdown-design.md`

## 核心要求

- 输出必须是“问题驱动拆书”，不要写成普通章节摘要
- 内容结论优先，简洁清楚，面向普通读者
- 不要伪造页码、引文、数据
- 如果细节不确定，使用保守概括
- HTML 不是简单粘贴 Markdown，而是重组为适合阅读的知识导航页

## Markdown 要求

生成一份“问题驱动拆书”的 Markdown，必须包含：

- 书名 / 英文名 / 作者 / 类型 / 核心主题 / 一句话概述
- 这本书试图回答什么问题
- 全书核心结论（3-6 条）
- 推荐怎么读这本书
- 全书问题地图（4-8 个核心问题，每个问题附一句话答案）
- 每个核心问题的详细拆解：
  - 一句话答案
  - 这部分在讲什么
  - 作者的核心逻辑
  - 关键证据 / 案例
  - 关键概念
  - 常见误解
  - 现实启发
  - 一句话记忆点
- 关键概念速查
- 常见误解 / 易混淆点
- 这本书对现实有什么帮助
- 一页看懂这本书

## HTML 要求

基于 Markdown 内容生成一个纯 HTML 单页，不使用 React / Vue / 构建工具。

要求：
- 可直接本地打开
- CSS 写在 `<style>` 中
- 少量 JS 写在 `<script>` 中
- 使用语义化 HTML
- 响应式，兼容桌面和手机
- 遵循 `design.md`

页面至少包含：
- Hero 区
- 全书核心结论区
- 问题地图区
- 详细拆解区
- 关键概念区
- 常见误解区
- 一页总结区
- 页面内导航
- 返回顶部
- 平滑滚动

## 页面风格

整体要像“清晰、可信、易读的知识产品页面”：
- 不像论文
- 不像营销页
- 有层级
- 有留白
- 方便扫读
- 适合长时间阅读

## 文件命名

先将书名转成合适的英文文件名：

- 用英文书名
- 全小写
- 空格改成 `-`
- 去掉不适合文件名的标点
- 如果原书名不是英文，优先使用常见官方英文名；没有官方英文名时，使用自然合理的英文译名
- 不要用拼音，除非没有更合适的英文名

示例：
- 《人类简史》 → `book-sapiens.md` / `book-sapiens.html`
- 《枪炮、病菌与钢铁》 → `book-guns-germs-and-steel.md` / `book-guns-germs-and-steel.html`
```
