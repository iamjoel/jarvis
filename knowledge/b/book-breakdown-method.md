# 拆书方法论
```md
拆书：{书名}。如果这本书在 `data/b/` 下已存在，则跳过这本书的拆书任务。

请完成以下任务：

1. 生成 `data/b/book-[english-name].md`。并在 `data/b/readme.md` 的拆书列表里添加链接和简介。
2. 如果需要生成 HTML，则生成 `pages/book-[english-name]/index.html`。纯 HTML 单页，不使用 React / Vue / 构建工具。设计参考: `pages/book-home-deus/index.html`。

## 核心要求

- 输出必须是“问题驱动拆书”，不要写成普通章节摘要
- 内容结论优先，简洁清楚，面向普通读者
- 不要伪造页码、引文、数据
- 如果细节不确定，使用保守概括
- HTML 不是简单粘贴 Markdown，而是重组为适合阅读的知识导航页

## Markdown 要求

生成一份“问题驱动拆书”的 Markdown，必须包含：

- 书名 / 英文名 / 作者 / 类型 / 核心主题 / 一句话概述
- 这本书试图回答什么问题
- 全书核心结论（3-6 条）
- 推荐怎么读这本书
- 全书问题地图（4-8 个核心问题，每个问题附一句话答案）
- 每个核心问题的详细拆解：
  - 一句话答案
  - 这部分在讲什么
  - 作者的核心逻辑
  - 关键证据 / 案例
  - 关键概念
  - 常见误解
  - 现实启发
  - 一句话记忆点
- 关键概念速查
- 常见误解 / 易混淆点
- 这本书对现实有什么帮助
- 一页看懂这本书

## 文件命名
先将书名转成合适的英文文件名：

- 用英文书名
- 全小写
- 空格改成 `-`
- 去掉不适合文件名的标点
- 如果原书名不是英文，优先使用常见官方英文名；没有官方英文名时，使用自然合理的英文译名
- 不要用拼音，除非没有更合适的英文名

示例：
- 《人类简史》 → `book-sapiens.md` / `book-sapiens.html`
- 《枪炮、病菌与钢铁》 → `book-guns-germs-and-steel.md` / `book-guns-germs-and-steel.html`
```
