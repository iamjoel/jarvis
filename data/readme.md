# Data

## 高密度摘要

- **一句话结论**：`data/` 保留“问题驱动”的总入口，但具体资料改由二级索引承接，避免首页继续膨胀。
- **核心机制**：先从本页选择问题域或横切主题，再进入二级索引中的问题表格，最后打开具体专题、拆书或资料。
- **判断入口**：不知道从哪里开始时先看“核心入口”；已知主题时直接进入健康、食物、书籍、技术、个人、社会或自然索引。
- **常见误区**：`a/` 到 `z/` 是存储分桶，不是阅读路径；不要按文件夹找知识，优先按问题索引进入。
- **相关文档**：[个人问题索引](./personal.md)、[社会问题索引](./society.md)、[自然问题索引](./nature.md)、[书籍索引](./books.md)、[knowledge/readme.md](../knowledge/readme.md)。

`data/` 按“问题”组织内容。顶层问题仍分为三类：

- 个人：人和自己的关系。
- 社会：人和人类社会的关系。
- 自然：人和自然的关系。

建议的使用方式：

1. 先从问题进入，而不是先从作者、学科或文件夹进入。
2. 先选本页入口，再顺着二级索引进入相关专题、拆书和资料。
3. 有价值的高频结论继续沉淀到 [knowledge](../knowledge/readme.md)。
4. 与解决问题有关的方法、自动化和 AI 工具统一进入 [技术索引](./technology.md)。

## 维护规则

1. 新增研究或资料文档时，优先写入 `data/` 下合适分桶。
2. 如果文档应被长期复用，把它挂到至少一个二级索引。
3. Markdown 文档应在标题后提供高密度摘要，说明一句话结论、核心机制、判断入口、常见误区和相关文档。
4. `data/readme.md` 只维护入口、路线图和全局说明；具体问题表格放到二级索引。
